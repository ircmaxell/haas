HAAS

Hugs As A Service

v0.0.1

Introduction

HAAS (Hugs As A Service) provides a modern, RESTful, scalable solution to the common problem of sending people virtual hugs.

Content Negotiation

HAAS will respond to the following 'Accept:' values with appropriate content

  • text/plain - Content will be returned as a plain string.
  • application/json - Content will be returned as a JSON object { message: 'message'}
  • text/html - Content will be returned as an HTML page with a twitter bootstrap hero unit, containing the message

API

/hug/:name/:from

Will return content of the form ":name, :from Sends You A HUG!!!", e.g. /hug/Sam/Bill will return 'Sam, Bill Sends You A HUG!!!'

/bearhug/:name/:from

Will return content of the form ":from Sends :name A Bear Hug!!!", e.g. /bearhug/Sam/Bill will return 'Bill Sends Sam A Bear Hug!!!'

/grouphug/:tolist/:fromlist

Will return content of the form ":tolist; :fromlist Send You A Hug!!!", parsing each argument as comma separated lists (adding commas and "and" as needed) e.g. /grouphug/Sam,Rich/Bill,Bob will return 'Sam and Rich; Bill and Bob Send You A HUG!!!'
